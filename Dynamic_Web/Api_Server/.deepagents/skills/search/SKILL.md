# Skill: Targeted Data Search (定向數據檢索與上傳分析)

此技能賦予 Shrimp-AI Researcher 存取專案核心數據庫與使用者暫存區的能力，作為 RAG 檢索與 AI Server 分析的前置步驟。

## 執行路徑規範 (Path Specification)

1. **多路徑監測範圍**: 
   - **知識庫路徑**: `../data/` (包含養殖手冊、物種規格、感測器日誌)。
   - **使用者上傳路徑**: `uploads/data/` (使用者透過前端傳送的大型影片、圖片或數據檔)。

2. **檔案存取約束 (Sandbox Rules)**:
   - **合法範圍**: 僅限於上述指定目錄。
   - **嚴禁行為**: 禁止存取任何涉及系統環境 (`.env`)、資料庫 (`user.db`) 或原始碼之路徑。

## 執行規範 (Guidelines)

1. **主動關聯邏輯 (Contextual Awareness)**:
   - 當使用者提到「這張圖」、「那段影片」或「分析我剛傳的東西」時，Agent 必須主動檢查 `uploads/data/` 下最新產生的檔案。
   - 針對大型檔案（如 1GB 影片），Agent **禁止** 讀取原始碼，必須改為調用「AI Video Analysis Server」進行處理。

2. **多物種檢索優先級**:
   - 優先搜尋 `../data/species/` 下對應種類的 `.md` 或 `.json` 規格文件。
   - 若為新上傳檔案，應先確認檔案名稱與使用者描述是否一致。

3. **路徑自動對齊 (Path Auto-Alignment)**:
   - 針對知識庫：自動補全 `../data/`。
   - 針對上傳區：自動補全 `uploads/data/`。

4. **數據預處理與摘要**:
   - **文字檔案**: 讀取前 500 字摘要與 Metadata（修改日期、大小）。
   - **多媒體檔案**: 僅檢索路徑與檔案名稱，隨後將其封裝成分析請求傳送至 AI 分析模組。

**Gotcha (注意事項)**：
- **大型檔案處理**: 遇到大型影片時，若直接執行 `read` 會導致 413 錯誤或系統崩潰。Agent 應僅定位路徑，並回報：「已定位影片檔案於 {path}，正在通知分析伺服器進行抽幀處理。」
- **路徑層級**: 確保 Python 運行的 `CWD` 能正確切換至父目錄 `../` 存取數據庫。
